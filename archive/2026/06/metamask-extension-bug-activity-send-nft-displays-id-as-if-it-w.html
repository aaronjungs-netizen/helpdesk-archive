Metamask Extension Bug Activity Send Nft Displays Id As If It Were Amount

Symptom

  • When attempting to send an NFT using the Metamask extension, the activity display shows the NFT's ID as if it were the amount being sent, instead of displaying the correct amount (usually 1, since NFTs are unique).

Technical Root Cause

  • This error occurs due to a bug in the Metamask extension's activity display logic, where it incorrectly interprets and displays the NFT's ID as the transaction amount.

Manual Overrides

  • Check the transaction details on the blockchain explorer to verify the correct amount.
  • Use the eth_getTransactionByHash RPC method to retrieve the transaction details and verify the amount.
  • Update the Metamask extension to the latest version using the following command:
npm install -g @metamask/core

Consider utilizing an external console for further debugging and troubleshooting.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum wallet set up with a valid account
  • NFTs available in the wallet for testing
