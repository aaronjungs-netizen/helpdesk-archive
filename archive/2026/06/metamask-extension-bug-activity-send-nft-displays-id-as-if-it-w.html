Metamask Extension Bug Activity Send Nft Displays Id As If It Were Amount

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem experience a bug when sending NFTs, where the activity log displays the NFT's ID as if it were the amount being sent, instead of showing the actual NFT being transferred.

Technical Root Cause

  • The error occurs due to a misinterpretation of the data type in the transaction payload, where the NFT's unique identifier is mistakenly treated as a numerical value representing the amount.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Check the transaction details on a blockchain explorer to verify the correct NFT transfer.
  • Use the Metamask API to manually construct and send the NFT transfer transaction.
  • Utilize a custom script to parse the transaction data and correct the display:
curl -X POST \
  https://api.metamask.io/transactions \
  -H 'Content-Type: application/json' \
  -d '{"from": "0x...","to": "0x...","value": "1","gas": "20000","gasPrice": "20000000000"}'
