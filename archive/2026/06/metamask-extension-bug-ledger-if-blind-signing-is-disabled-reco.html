Metamask Extension Bug Ledger If Blind Signing Is Disabled Reconnect

Symptom

  • Users encounter an error when attempting to connect their Ledger device to Metamask, with blind signing disabled, resulting in a prompt to reconnect from the beginning of the workflow.

Technical Root Cause

  • The error occurs due to a compatibility issue between Metamask's extension and the Ledger device when blind signing is disabled, causing the connection process to fail and prompting the user to restart the workflow.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Enable blind signing on the Ledger device and retry the connection process
  • Update Metamask to the latest version
  • Use the following JSON configuration to override the default settings:
{
  "ledger": {
    "blindSigning": true
  }
}

Consider using an external console to execute these commands for a more streamlined experience.

Prerequisites

  • Metamask extension installed and up-to-date
  • Ledger device properly configured and connected
  • Blind signing disabled on the Ledger device
