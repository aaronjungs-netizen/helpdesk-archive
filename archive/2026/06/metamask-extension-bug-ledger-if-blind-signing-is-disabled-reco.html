Metamask Extension Bug Ledger If Blind Signing Is Disabled Reconnect

Symptom

  • Users encounter an error when attempting to connect their Ledger device to Metamask, specifically when blind signing is disabled, resulting in a prompt to reconnect from the beginning of the workflow.

Technical Root Cause

  • The error occurs due to a compatibility issue between Metamask's extension and Ledger's device settings, particularly when blind signing is disabled, causing the connection process to fail.

Manual Overrides

  • To resolve this issue, try the following manual overrides:
  • Enable blind signing on the Ledger device and then reconnect to Metamask.
  • Update Metamask and Ledger Live to the latest versions.
  • Use the following JSON configuration to manually set up the connection:
{
  "ledger": {
    "blindSigning": true
  }
}

Consider utilizing an external console for a more streamlined debugging experience.

Prerequisites

  • Metamask extension installed and updated to the latest version.
  • Ledger device firmware updated to the latest version.
  • Blind signing enabled or disabled according to the user's preference.
