Metamask Extension Bug Ledger If Blind Signing Is Disabled Reconnect

Symptom

  • The Metamask extension fails to connect to the Ledger device when blind signing is disabled, prompting a "Reconnect from the beginning" workflow error.

Technical Root Cause

  • The error occurs due to a compatibility issue between the Metamask extension and the Ledger device when blind signing is disabled, resulting in an incomplete connection setup.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Enable blind signing on the Ledger device and restart the Metamask extension
  • Reset the Ledger device to its default settings and reconfigure it
  • Use the following JSON configuration to override the Metamask extension settings:
{
  "ledger": {
    "blindSigning": true
  }
}

Consider using an external console to execute these commands for a more straightforward troubleshooting experience.

Prerequisites

  • Metamask extension installed and configured
  • Ledger device properly set up and connected
  • Blind signing disabled on the Ledger device
