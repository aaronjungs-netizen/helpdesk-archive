Metamask Nonce Out Of Sync

Symptom

  • Users experience transaction failures with an error message indicating that their MetaMask nonce is out of sync, preventing them from successfully sending Ethereum transactions.

Technical Root Cause

  • The error occurs when the transaction nonce stored in MetaMask does not match the nonce expected by the Ethereum network, often due to cancelled, pending, or stuck transactions that have incremented the nonce without being confirmed.

Manual Overrides

  • Reset the account in MetaMask to try and synchronize the nonce.
  • Manually adjust the nonce value in the transaction settings to match the expected nonce.
  • Use the following JSON-RPC method to reset the account:
{
  "method": "eth_accounts",
  "params": [],
  "id": 1,
  "jsonrpc": "2.0"
}

Consider utilizing an external console for more advanced debugging and nonce management.

Prerequisites

  • Ensure MetaMask is updated to the latest version.
  • Verify the Ethereum network connection is stable.
  • Check for any pending or stuck transactions that may be causing the nonce discrepancy.
