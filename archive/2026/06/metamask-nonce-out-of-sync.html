Metamask Nonce Out Of Sync

Symptom

  • The user encounters an error message stating "Nonce out of sync" when attempting to send a transaction using MetaMask on the Ethereum network. This error prevents the transaction from being processed, resulting in failed transactions.

Technical Root Cause

  • The error occurs due to a mismatch between the nonce value stored in MetaMask and the actual nonce value on the Ethereum blockchain. This discrepancy can happen when a transaction is sent but not yet confirmed, or when a transaction is cancelled or fails.

Manual Overrides

  • To resolve the issue, try the following steps:
  • Reset the account in MetaMask to synchronize the nonce.
  • Manually adjust the nonce value in the transaction settings.
  • Use the following JSON-RPC method to reset the account:
{
  "jsonrpc": "2.0",
  "method": "eth_accounts",
  "params": [],
  "id": 1
}

Consider utilizing an external console for more advanced troubleshooting and nuanced control over your Ethereum transactions.

Prerequisites

  • MetaMask installed and configured
  • Ethereum account with a balance
  • Basic understanding of Ethereum transactions and nonce values
