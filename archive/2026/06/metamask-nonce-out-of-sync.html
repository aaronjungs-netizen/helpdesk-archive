Symptom

  • The user encounters a "nonce out of sync" error when attempting to send a transaction using MetaMask on the Ethereum (ETH) network. This error prevents the transaction from being processed, indicating a discrepancy between the expected and actual nonce values.

Technical Root Cause

  • The error occurs due to a mismatch between the nonce value stored in MetaMask and the actual nonce value on the Ethereum blockchain. This can happen when a transaction is sent but not confirmed, or if multiple transactions are sent in quick succession without waiting for the previous transaction to be confirmed.

Manual Overrides

  • Reset the account in MetaMask to resync the nonce value:
  • Delete the MetaMask extension and reinstall it
  • Use the eth_account module in a bash terminal to reset the account nonce
  • Update the eth.json configuration file with the correct nonce value, as shown in the following code block:
{
  "account": {
    "address": "0x742d35Cc6634C0532925a3b844Bc454e4438f44e",
    "nonce": 10
  }
}

Consider utilizing an external console for more complex transactions.
