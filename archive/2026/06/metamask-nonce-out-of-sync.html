Symptom

  • The user encounters an error message "MetaMask nonce out of sync" when attempting to send a transaction on the Ethereum network, indicating that the transaction nonce stored in MetaMask does not match the nonce on the Ethereum blockchain.

Technical Root Cause

  • This error occurs due to a discrepancy between the expected transaction count (nonce) in MetaMask and the actual transaction count on the Ethereum blockchain, often resulting from cancelled or pending transactions that have not been properly accounted for.

Manual Overrides

  • Reset the account in MetaMask to resynchronize the nonce with the blockchain.
  • Use the Ethereum blockchain explorer to verify the current transaction count for the specific account and manually adjust the nonce in MetaMask if necessary.
  • Utilize the following JSON-RPC method to reset the account:
{
  "method": "eth_accounts",
  "params": [],
  "id": 1,
  "jsonrpc": "2.0"
}

Consider utilizing an external interface for more complex JSON-RPC commands.

Prerequisites

  • MetaMask extension installed and configured.
  • Ethereum account with a history of transactions.
  • Access to the Ethereum blockchain explorer (e.g., Etherscan).
