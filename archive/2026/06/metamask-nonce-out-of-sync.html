Metamask Nonce Out Of Sync

Symptom

  • The user encounters an error message indicating that MetaMask's nonce is out of sync, typically when attempting to send a transaction on the Ethereum network. This results in failed transactions and prevents the user from interacting with smart contracts or sending ETH.

Technical Root Cause

  • The error occurs when the nonce (a counter tracking the number of transactions sent from an address) stored in MetaMask does not match the nonce on the Ethereum blockchain. This discrepancy can happen due to pending transactions, transaction failures, or manually adjusting the nonce.

Manual Overrides

  • To resolve the issue, try the following steps:
  • Reset the MetaMask account by removing and re-adding it.
  • Manually adjust the nonce in the transaction settings to match the current nonce on the blockchain.
  • Use the following JSON-RPC method to get the current nonce:
{
  "jsonrpc": "2.0",
  "method": "eth_getTransactionCount",
  "params": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e", "latest"],
  "id": 1
}

Consider using an external Ethereum console like Etherscan to verify the current nonce and transaction history.

Prerequisites

  • Ethereum network connection
  • MetaMask installed and configured
  • Basic understanding of Ethereum transactions and nonces
