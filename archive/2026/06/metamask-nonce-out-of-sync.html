Metamask Nonce Out Of Sync

Symptom

  • Users encounter an error message "MetaMask nonce out of sync" when attempting to send a transaction on the Ethereum network, indicating that the transaction nonce stored in MetaMask does not match the nonce expected by the Ethereum blockchain.

Technical Root Cause

  • This error occurs due to a mismatch between the transaction count (nonce) stored locally in MetaMask and the actual transaction count on the Ethereum blockchain. This discrepancy can happen due to various reasons such as failed or pending transactions that were not properly accounted for.

Manual Overrides

  • Reset the account in MetaMask to resynchronize the nonce:
  • Go to the MetaMask settings and select "Advanced"
  • Click on "Reset Account" to reset the account and resynchronize the nonce
  • Use the following command to manually set the nonce (if necessary):
eth_sendTransaction { 
  "from": "0x...your_address...", 
  "nonce": "0x...correct_nonce...", 
  "gas": "0x...", 
  "gasPrice": "0x..." 
}

Consider utilizing an external console for more complex transactions and precise control.

Prerequisites

  • MetaMask extension installed and configured
  • Ethereum account with a sufficient balance
  • Access to the Ethereum network via a node or provider like Infura
