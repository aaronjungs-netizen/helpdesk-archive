Symptom

  • The user encounters an error message stating "MetaMask nonce out of sync" when attempting to send a transaction on the Ethereum network using MetaMask.

Technical Root Cause

  • This error occurs when the nonce (a counter that keeps track of the number of transactions sent from an address) stored in MetaMask does not match the nonce on the Ethereum blockchain, causing a mismatch and preventing transactions from being processed.

Manual Overrides

  • Reset the account in MetaMask to resynchronize the nonce with the blockchain:
  • Delete the MetaMask extension and reinstall it
  • Use the MetaMask interface to reset the account
  • Manually update the nonce using the following command:
eth_sendTransaction { 
  "from": "0x...your_address",
  "nonce": "0x...correct_nonce"
}

Consider utilizing an external interface for more complex overrides.

Prerequisites

  • MetaMask extension installed and configured
  • Ethereum wallet with a balance
  • Access to the Ethereum blockchain via a node or provider
