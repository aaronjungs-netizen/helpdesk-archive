Symptom

  • Users encounter an error message "MetaMask nonce out of sync" when attempting to send a transaction on the Ethereum network, indicating a discrepancy between the expected and actual nonce values.

Technical Root Cause

  • The error occurs due to a mismatch between the account nonce stored in MetaMask and the actual nonce on the Ethereum blockchain, often resulting from concurrent transactions or wallet resets.

Manual Overrides

  • Reset the account nonce in MetaMask to match the on-chain value:
  • Update the MetaMask settings to reflect the correct nonce
  • Use the Ethereum blockchain explorer to verify the correct nonce value
  • Utilize the following code to reset the nonce:
eth.accounts[0].nonce = eth.getTransactionCount(eth.accounts[0])

Prerequisites

  • MetaMask extension installed and configured
  • Ethereum wallet with a sufficient balance
