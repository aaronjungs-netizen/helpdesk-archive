Symptom

  • The user encounters a "Nonce out of sync" error when attempting to send a transaction using MetaMask on the Ethereum (ETH) network, indicating that the transaction nonce (a sequential number) stored in MetaMask does not match the nonce expected by the Ethereum network.

Technical Root Cause

  • This error occurs because the nonce is used to prevent transaction replay attacks, and when it gets out of sync, the Ethereum network rejects transactions with non-sequential nonces, causing this error.

Manual Overrides

  • Reset the account in MetaMask to resync the nonce:
  • Go to MetaMask settings
  • Select the affected account
  • Click on "Reset Account"
  • Use the Ethereum JSON-RPC eth_getTransactionCount method to manually check the current nonce:
curl -X POST \
  http://localhost:8545 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"eth_getTransactionCount","params":["0x742d35Cc6634C0532925a3b844Bc454e4438f44e","latest"],"id":1}'
  • Manually set the nonce in the transaction options before sending a transaction

Prerequisites: - MetaMask installed and configured - Ethereum node (like Geth) running locally - Familiarity with JSON-RPC and transaction nonces

Consider using an external console like MetaMask's built-in console for easier debugging.
