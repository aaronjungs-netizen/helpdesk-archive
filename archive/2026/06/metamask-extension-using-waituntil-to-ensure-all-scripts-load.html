Metamask Extension Using Waituntil To Ensure All Scripts Load

Symptom

  • Users experience delayed or failed loading of web pages when using the Metamask extension, resulting in a poor browsing experience.

Technical Root Cause

  • The Metamask extension uses the waitUntil protocol to ensure that all necessary scripts are loaded before allowing the webpage to proceed, which can sometimes cause delays if the scripts take too long to load.

Manual Overrides

  • To resolve this issue, users can try the following manual overrides:
  • Disable and re-enable the Metamask extension to restart the loading process
  • Clear the browser cache to remove any potentially corrupted data
  • Use the following code to manually override the waitUntil protocol:
mm_waitUntil_override=true

For advanced users, consider using an external console for more granular control over the loading process.

Prerequisites

  • Metamask extension installed and enabled
  • Basic understanding of browser extensions and their functionality
  • Familiarity with command-line interfaces and scripting languages
