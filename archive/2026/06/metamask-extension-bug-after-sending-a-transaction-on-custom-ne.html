Metamask Extension Bug After Sending A Transaction On Custom Network

Symptom

  • Users experience a disappearance of their balance on the Metamask extension after sending a transaction on a custom network, despite the transaction being successfully processed on the blockchain.

Technical Root Cause

  • The issue arises due to a synchronization problem between the Metamask extension and the custom network, leading to an incorrect update of the user's balance.

Manual Overrides

  • Attempt to refresh the Metamask extension manually to re-sync the balance.
  • Use the eth_getBalance RPC method to query the balance directly from the custom network node.
  • Update the Metamask configuration to include the custom network's chain ID and RPC URL:
{
  "chainId": 12345,
  "rpcUrl": "https://custom-network-node.com"
}

Consider utilizing an external console for further debugging and direct access to the Ethereum network.

Prerequisites

  • Metamask extension installed and configured.
  • Custom network added to the Metamask extension.
  • A basic understanding of Ethereum RPC methods and network configuration.
