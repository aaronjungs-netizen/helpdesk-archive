Metamask Extension Bug After Sending A Transaction On Custom Network

Symptom

  • The user observes that their balance disappears from the Metamask extension after sending a transaction on a custom network, despite the transaction being successfully processed on the blockchain.

Technical Root Cause

  • The issue arises due to a bug in the Metamask extension that fails to update the balance correctly after transactions on custom networks, leading to a discrepancy between the actual and displayed balances.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Refreshing the Metamask extension to attempt a rebalance
  • Checking the custom network's blockchain explorer to verify the transaction and balance
  • Using the following JSON-RPC command to manually update the balance:
{
  "jsonrpc": "2.0",
  "method": "eth_getBalance",
  "params": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e", "latest"],
  "id": 1
}

Prerequisites: * Metamask extension installed and configured * Custom network added to Metamask * Transaction sent on the custom network For further debugging, consider utilizing an external console to execute these commands.
