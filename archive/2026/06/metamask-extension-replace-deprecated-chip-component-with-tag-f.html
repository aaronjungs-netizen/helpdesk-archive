Symptom

  • The user sees an error message indicating that the Chip component is deprecated and needs to be replaced with the Tag component from the component-library in the Metamask extension.

Technical Root Cause

  • The error occurs because the Chip component is no longer supported and has been replaced by the Tag component, which provides more flexibility and customization options.

Manual Overrides

  • Update the import statement to use the Tag component:
  • Replace import { Chip } from '@material-ui/core'; with import { Tag } from '@component-library/tag';
  • Update the Chip component usage to use the Tag component
  • Remove any deprecated Chip component props and replace with Tag component props
npm install @component-library/tag

Consider using an external console to verify the installation of the new package.

Prerequisites

  • Metamask extension installed
  • Node.js and npm installed
  • Access to the component-library documentation
