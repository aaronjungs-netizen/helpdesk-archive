Symptom

  • The Metamask extension is displaying an error message indicating that the Chip component is deprecated and needs to be replaced with the Tag component from the component-library.

Technical Root Cause

  • The error occurs because the Chip component is no longer supported in the latest version of the component-library, and the Metamask extension is still using the outdated component.

Manual Overrides

  • Update the component-library to the latest version using npm install @component-library/latest
  • Replace all instances of the Chip component with the Tag component in the Metamask extension code
  • Configure the Tag component to match the existing Chip component functionality, for example:
{
  "components": {
    "Tag": {
      "variant": "filled",
      "color": "primary"
    }
  }
}

Consider utilizing an external console to execute these commands for a more streamlined experience.

Prerequisites

  • Node.js installed on the system
  • npm package manager installed and configured
  • Metamask extension code repository cloned and accessible
