Metamask Extension Bug Trying To Navigate Back From The Manage Network Page

Symptom

  • When attempting to navigate back from the Manage Network page in Metamask, the user is displayed an incorrect page or an error message, instead of being redirected to the expected previous page.

Technical Root Cause

  • This error occurs due to a bug in the Metamask extension's navigation handling, specifically when transitioning from the Manage Network page. The extension fails to properly track the navigation history, resulting in an incorrect page being displayed when the user attempts to navigate back.

Manual Overrides

  • To resolve this issue, users can try the following manual overrides:
  • Clearing the browser's cache and cookies
  • Disabling and re-enabling the Metamask extension
  • Using the following command to reset the extension's navigation state:
localStorage.clear("metamask_navigation_state")

Consider accessing the browser's developer console for further troubleshooting.

Prerequisites

  • Metamask extension installed and enabled
  • Browser cache and cookies cleared
  • Familiarity with browser developer console usage
