Metamask Extension Bug Unable To Cancel Pending Transaction Due

Symptom

  • Users are unable to cancel pending transactions in Metamask, resulting in an error message indicating that there is not enough gas to perform the cancellation.

Technical Root Cause

  • The error occurs because the pending transaction has a higher gas price than the cancellation transaction, causing the cancellation transaction to be stuck in the mempool.

Manual Overrides

  • Increase the gas price of the cancellation transaction to override the pending transaction:
  • Use a higher gas price than the pending transaction
  • Utilize the eth_sendTransaction JSON-RPC method to send a custom transaction
  • Modify the transaction using the following code block:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...your_address...",
      "to": "0x...contract_address...",
      "gas": "0x...gas_limit...",
      "gasPrice": "0x...higher_gas_price..."
    }
  ],
  "id": 1
}

Consider using an external console to execute the above command for more flexibility.
