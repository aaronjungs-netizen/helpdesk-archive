Metamask Extension Bug: Unable To Cancel Pending Transaction Due To Not Enough Gas Error

Symptom

  • Users are unable to cancel pending transactions in Metamask, resulting in an error message indicating that there is not enough gas.

Technical Root Cause

  • This error occurs due to a bug in the Metamask extension, where the gas estimation for canceling a pending transaction is incorrect, leading to an insufficient gas error.

Manual Overrides

  • Increase the gas limit manually by using the following JSON RPC method: 
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...address",
      "to": "0x...contract",
      "gas": "0x...increased gas limit"
    }
  ]
}
  • Use the eth_estimateGas method to get a more accurate gas estimate before attempting to cancel the transaction.
  • Manually set the gas price to a higher value to prioritize the cancellation transaction.

Prerequisites: * Metamask extension installed and configured * Sufficient ETH balance to cover gas costs * Basic understanding of JSON RPC methods

Consider utilizing an external Ethereum console to execute these commands for more flexibility.
