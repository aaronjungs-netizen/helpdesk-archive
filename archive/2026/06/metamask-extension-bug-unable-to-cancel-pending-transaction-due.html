Metamask Extension Bug Unable To Cancel Pending Transaction Due

Symptom

  • Users are unable to cancel pending transactions using the Metamask extension, resulting in an error message indicating that there is not enough gas.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that prevents it from properly handling the cancellation of pending transactions, specifically when the gas price is set too low.

Manual Overrides

  • Increase the gas price to a higher value to ensure the transaction can be processed and then cancelled.
  • Use the eth_cancelTransaction method to manually cancel the pending transaction.
  • Update the transaction with a new gas price using the following code block:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...your_address...",
      "to": "0x...contract_address...",
      "gas": "0x...gas_limit...",
      "gasPrice": "0x...new_gas_price..."
    }
  ],
  "id": 1
}

For optimal results, consider using an external console to execute these commands.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum node or provider (e.g. Infura) set up
  • Sufficient ETH balance to cover transaction fees
