Metamask Extension Bug Unable To Cancel Pending Transaction Due

Symptom

  • Users are unable to cancel pending transactions using the Metamask extension, resulting in an error message indicating that there is not enough gas to perform the cancellation.

Technical Root Cause

  • The error occurs when the pending transaction's gas price is higher than the current network gas price, causing the cancellation transaction to require more gas than is available, thus failing due to insufficient gas.

Manual Overrides

  • Increase the gas limit for the cancellation transaction to ensure it has enough gas to override the pending transaction.
  • Use a higher gas price to expedite the cancellation transaction and increase its priority in the network.
  • Manually construct and send a replacement transaction with a higher gas price and sufficient gas to cancel the pending transaction, as shown in the following JSON-RPC example:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [

      "from": "0x...your_address...",

      "gas": "0x...gas_amount...",
      "gasPrice": "0x...gas_price...",
      "nonce": "0x...nonce..."
    }
  ],
  "id": 1
}
