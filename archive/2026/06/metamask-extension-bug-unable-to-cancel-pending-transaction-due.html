Metamask Extension Bug Unable To Cancel Pending Transaction Due

Symptom

  • Users are unable to cancel pending transactions using the Metamask extension, resulting in a "not enough gas" error message. The transaction remains in a pending state, and attempts to cancel or speed up the transaction are unsuccessful.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that prevents it from properly handling gas management for pending transactions. This can lead to the extension being unable to allocate sufficient gas to cancel the pending transaction.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Increase the gas limit manually using the Metamask interface
  • Use a higher gas price to prioritize the cancellation transaction
  • Utilize the following JSON-RPC method to cancel the pending transaction:
{
  "jsonrpc": "2.0",
  "method": "eth_cancelTransaction",
  "params": [
    {
      "from": "0x...user_address...",
      "to": "0x...contract_address...",
      "gas": "0x...gas_limit...",
      "gasPrice": "0x...gas_price..."
    }
  ],
  "id": 1
}

Consider using an external console to execute the JSON-RPC method for more flexibility.

Prerequisites

  • Metamask extension installed and configured
  • Sufficient ETH balance to cover transaction fees
  • Basic understanding of JSON-RPC methods and gas management in Ethereum transactions
