Metamask Extension Bug Incorrect Balance For Aave V3 Wavax Token

Symptom

  • Users of the Metamask extension on the Avalanche network may observe an incorrect balance display for the Aave v3 WAVAX token, leading to potential confusion and incorrect transaction decisions.

Technical Root Cause

  • The issue arises due to a synchronization problem between the Metamask extension and the Aave v3 protocol on the Avalanche network, resulting in outdated or incorrect token balance information being displayed to the user.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Refresh the Metamask extension and retry fetching the token balance.
  • Manually update the token contract address in the Metamask extension to ensure it matches the latest Aave v3 WAVAX token contract.
  • Utilize the following JSON payload to update the token information via the Metamask API:
{
  "jsonrpc": "2.0",
  "method": "eth_call",
  "params": [
    {
      "to": "0x...AaveV3WAVAXContractAddress...",
      "data": "0x...getTokenBalanceData..."
    },
    "latest"
  ],
  "id": 1
}

Consider using an external console for more complex token balance queries.

Prerequisites

  • Metamask extension installed and configured on the Avalanche network
  • Aave v3 WAVAX token added to the Metamask extension
  • Latest version of the Metamask extension and Aave v3 protocol updates applied
