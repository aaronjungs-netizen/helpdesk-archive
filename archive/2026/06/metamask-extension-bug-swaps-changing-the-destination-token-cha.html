Metamask Extension Bug Swaps Changing The Destination Token Cha

Symptom

  • When attempting to change the destination token in a swap transaction on the Metamask extension, the source token is changed instead, resulting in an unexpected transaction setup.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's swap functionality, where the token selection inputs are incorrectly referenced, causing the source token to be modified when attempting to change the destination token.

Manual Overrides

  • To resolve this issue, users can attempt the following manual overrides:
  • Refresh the Metamask extension and retry the swap transaction.
  • Manually input the destination token's contract address to bypass the token selection interface.
  • Use the following JSON configuration to override the token selection:
{
  "sourceToken": "0x...source_token_address...",
  "destinationToken": "0x...destination_token_address..."
}

Consider using an external console to verify the transaction details before confirming.
