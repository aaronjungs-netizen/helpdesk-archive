Metamask Extension Bug Swaps Changing The Destination Token Cha

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem may encounter a bug where swapping tokens results in an unexpected behavior: when attempting to change the destination token in a swap, the source token is modified instead.

Technical Root Cause

  • This error occurs due to a logical inconsistency in the Metamask extension's swap functionality, where the input fields for source and destination tokens are not properly isolated, leading to unintended token swaps.

Manual Overrides

  • To manually correct this issue, users can attempt the following steps:
  • Refresh the Metamask extension and retry the swap.
  • Clear the Metamask extension's storage and restart the browser.
  • Use the following JSON configuration to reset the swap settings:
{
  "sourceToken": "ETH",
  "destinationToken": "DAI",
  "amount": "1.0"
}

Consider utilizing an external console for more complex token swap operations.

Prerequisites

  • Metamask extension installed and configured.
  • Ethereum (ETH) wallet with sufficient balance.
  • Stable internet connection for seamless transactions.
