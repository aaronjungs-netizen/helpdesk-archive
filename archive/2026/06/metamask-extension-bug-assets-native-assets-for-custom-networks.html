Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selecting them in the Metamask extension, resulting in an incomplete display of available assets.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's asset handling mechanism, specifically when dealing with custom networks. The extension fails to properly fetch and display native assets for these networks.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the asset list by deleting the assetList.json file
  • Use a custom configuration to force asset refresh:
{
  "network": "custom",
  "nativeAsset": "ETH",
  "assetList": []
}

Consider using an external console to execute these commands for a more streamlined experience.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • Latest version of the Metamask extension installed
