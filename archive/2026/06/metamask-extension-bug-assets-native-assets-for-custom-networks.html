Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selecting them in the Metamask extension, despite having added the custom network and assets correctly.

Technical Root Cause

  • The issue arises due to a bug in the Metamask extension's asset handling mechanism for custom networks, where it fails to properly fetch and display native assets.

Manual Overrides

  • To resolve this issue, users can attempt the following manual overrides:
  • Update the Metamask extension to the latest version
  • Manually add the native asset using the asset's contract address and decimal places
  • Use a custom configuration file to override the asset display settings
{
  "network": "custom-network",
  "nativeAsset": {
    "address": "0x...contract-address...",
    "decimals": 18
  }
}

For further debugging, consider utilizing an external console to inspect the extension's internal state.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the extension
  • Native asset contract address and decimal places known
