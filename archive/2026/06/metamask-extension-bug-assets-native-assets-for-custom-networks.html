Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selected in the Metamask extension, despite proper configuration.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension's handling of custom network configurations, specifically with regards to native asset display.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Update the Metamask extension to the latest version
  • Manually add the native asset token to the custom network configuration using the following JSON configuration:
{
  "chainId": "0x${CHAIN_ID}",
  "nativeCurrency": {
    "name": "${NATIVE_CURRENCY_NAME}",
    "symbol": "${NATIVE_CURRENCY_SYMBOL}",
    "decimals": ${NATIVE_CURRENCY_DECIMALS}
  }
}
  • Clear the Metamask extension's storage and restart the browser Consider using an external console for more advanced troubleshooting and configuration adjustments.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • Native asset token properly configured for the custom network
