Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selected in the Metamask extension, resulting in incomplete or incorrect asset displays.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension's handling of custom network configurations, specifically with regards to native asset recognition and display.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the Metamask extension's network settings to default
  • Utilize the following JSON configuration to manually specify native assets for custom networks:
{
  "networks": [
    {
      "name": "Custom Network",
      "nativeAsset": "ETH"
    }
  ]
}

Consider utilizing an external console for more advanced configuration and troubleshooting.

Prerequisites

  • Metamask extension installed and enabled
  • Custom network configured in the Metamask extension
  • Latest version of the Metamask extension installed
