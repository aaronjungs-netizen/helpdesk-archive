Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • When selecting a custom network in Metamask, native assets are not displayed, leading to difficulties in managing and transferring assets on these networks.

Technical Root Cause

  • The error occurs due to Metamask's inability to automatically detect and list native assets for custom networks, resulting in an incomplete asset list.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Update Metamask to the latest version to ensure compatibility with custom networks.
  • Manually add the native asset token address and details for the custom network.
  • Utilize the following JSON configuration to override default settings:
{
  "network": "custom-network",
  "nativeAsset": {
    "address": "0x...native-asset-token-address",
    "symbol": "CUSTOM",
    "decimals": 18
  }
}

Consider using an external console for more complex configuration adjustments.

Prerequisites

  • Metamask extension installed and updated
  • Custom network added to Metamask
  • Native asset token address and details available
