Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users experience an issue where removing a hardware wallet account from Metamask does not clear the associated permissions, resulting in persistent access and potential security risks.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the removal of a hardware wallet account does not properly revoke the previously granted permissions, leading to a mismatch between the user's intended access settings and the actual permissions stored by the extension.

Manual Overrides

  • Remove the affected account and reinstall the Metamask extension
  • Use the Metamask API to manually revoke permissions for the removed account
  • Execute the following command to clear the permission cache:
mm-cli --revoke-permissions --account <account_address>

Consider utilizing an external console for more advanced troubleshooting and configuration.

Prerequisites

  • Metamask extension installed
  • Hardware wallet connected
  • Basic understanding of command-line interfaces
