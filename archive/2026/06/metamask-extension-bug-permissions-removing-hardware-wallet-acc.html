Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users are experiencing an issue where removing a hardware wallet account from Metamask does not clear the associated permissions, resulting in persistent access and potential security risks.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the permissions are not properly updated when a hardware wallet account is removed, leading to a mismatch between the user's intent and the actual access control.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Delete the Metamask extension and reinstall it
  • Reset the Metamask account by deleting the local storage and clearing the browser cache
  • Update the permissions manually using the following JSON configuration:
{
  "permissions": {
    "hardwareWallets": []
  }
}

For advanced users, consider utilizing a terminal to inspect and modify the configuration. Prerequisites: * Basic understanding of JSON configuration * Familiarity with Metamask extension settings * Access to the browser's developer tools
