Metamask Extension Bug Account Discovery And Syncs Error For Social Login Import Users

Symptom

  • Users encounter an error when attempting to import their accounts using social login, resulting in a failure to discover and sync their accounts within the Metamask extension.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that prevents it from correctly handling the account discovery and syncing process for social login import users, likely caused by an issue with the extension's internal state management or API request handling.

Manual Overrides

  • Reset Metamask extension settings to default: mm-reset
  • Clear browser storage for Metamask: 
{
  "clearStorage": true,
  "storageKeys": ["metamask", "eth_accounts"]
}
  • Re-initialize the Metamask extension: mm-init

Prerequisites: * Metamask extension installed and enabled * Social login credentials correctly configured * Ethereum network selected and connected

Consider using an external console to execute the manual override commands for more flexibility.
