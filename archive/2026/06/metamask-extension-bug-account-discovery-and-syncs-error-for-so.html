Metamask Extension Bug Account Discovery And Syncs Error For Social Login Import Users

Symptom

  • Users encounter an error when attempting to import their accounts using social login, resulting in a failed account discovery and sync process within the Metamask extension.

Technical Root Cause

  • The error occurs due to a mismatch between the expected and actual data formats during the social login import process, causing the Metamask extension to fail in syncing the user's accounts.

Manual Overrides

  • Attempt to re-initialize the social login import process by running the following commands:
  • mm-reset-social-login
  • mm-reinit-account-sync
  • mm-force-account-discovery
{
  "socialLogin": {
    "import": {
      "accountDiscovery": true,
      "accountSync": true
    }
  }
}

Consider utilizing an external console to troubleshoot the issue further for more complex scenarios.

Prerequisites

  • Metamask extension installed and enabled
  • Social login credentials configured correctly
  • Ethereum network selected in the Metamask extension
