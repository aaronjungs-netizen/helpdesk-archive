Metamask Extension Bug Confirmations Security Alerts Error Validation

Symptom

  • Users encounter an error message stating "Error validating request with PPOM con" when attempting to confirm transactions or interact with security alerts within the Metamask extension on the Ethereum (ETH) ecosystem.

Technical Root Cause

  • This error occurs due to a bug in the Metamask extension that affects the validation process of requests, potentially related to the Proof of Work (PoW) or Proof of Stake (PoS) consensus mechanisms, or issues with the extension's connection to the Ethereum network.

Manual Overrides

  • To troubleshoot this issue, try the following manual overrides:
  • Restart the browser and reload the Metamask extension.
  • Clear the browser's cache and cookies related to the Metamask extension.
  • Execute the following JSON payload to reset the extension's configuration:
{
  "reset": true,
  "config": {
    "network": "mainnet",
    "provider": "https://mainnet.infura.io/v3/YOUR_PROJECT_ID"
  }
}

Consider utilizing an external console for more complex troubleshooting procedures.

Prerequisites

  • Ensure you have the latest version of the Metamask extension installed.
  • Verify that your Ethereum wallet is properly connected to the Metamask extension.
  • Check for any conflicting browser extensions that may interfere with Metamask's functionality.
