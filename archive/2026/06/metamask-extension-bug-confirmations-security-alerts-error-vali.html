Metamask Extension Bug Confirmations Security Alerts Error Validation

Symptom

  • Users encounter an error message "Error validating request" when attempting to confirm transactions or access security alerts within the Metamask extension, specifically related to PPOM (Pending Parent Outsized Multi) confirmations.

Technical Root Cause

  • The error occurs due to a mismatch between the expected and actual transaction structures during the validation process, potentially caused by outdated extension versions, incorrect network configurations, or issues with the PPOM mechanism.

Manual Overrides

  • Update Metamask extension to the latest version to ensure compatibility with the latest Ethereum network updates.
  • Verify that the Ethereum network is correctly configured and set to the intended chain ID.
  • Utilize the following JSON configuration to reset the PPOM validation parameters:
{
  "ppomValidation": {
    "enabled": true,
    "timeout": 30000
  }
}

Consider utilizing an external console for further debugging and customization of your Metamask configuration.

Prerequisites

  • Metamask extension installed and configured on your browser.
  • Ethereum wallet set up and connected to the Metamask extension.
  • Basic understanding of Ethereum network configurations and transaction structures.
