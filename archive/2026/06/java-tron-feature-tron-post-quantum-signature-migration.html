TRON Post-Quantum Signature Migration Failure

Symptom

  • Users encounter an error when attempting to migrate to post-quantum signatures on the TRON network, resulting in failed transactions and inability to access their accounts.

Technical Root Cause

  • The error occurs due to incomplete or incorrect configuration of the post-quantum signature migration process, leading to a mismatch between the user's wallet and the TRON network's expectations.

Manual Overrides

  • Update the config.conf file with the correct post-quantum signature parameters:
{
  "postQuantum": {
    "enabled": true,
    "signatureScheme": "SPHINCS"
  }
}
  • Restart the Java Tron node to apply the changes: java -jar java-tron.jar --reset
  • Verify the post-quantum signature migration status using the TRON Web wallet or CLI tool

Consider utilizing an external console for more complex configurations.

Prerequisites

  • Java Tron node installed and running
  • TRON wallet with post-quantum signature support
  • Latest version of the Java Tron software
