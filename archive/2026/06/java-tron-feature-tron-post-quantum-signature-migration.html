TRON Post-Quantum Signature Migration Failure

Symptom

  • Users experience errors when attempting to migrate to TRON's post-quantum signature scheme, resulting in failed transactions and inability to interact with the network.

Technical Root Cause

  • The error occurs due to incomplete or incorrect configuration of the post-quantum signature parameters in the java-tron implementation, leading to a mismatch between the expected and actual signature formats.

Manual Overrides

  • Update the tron.conf file to include the correct post-quantum signature parameters:
  • Set postQuantumSignatureEnabled to true
  • Configure the postQuantumSignatureScheme to the desired algorithm (e.g., SPHINCS+)
  • Run the following command to update the configuration:
java -jar java-tron.jar --post-quantum-signature-enabled true --post-quantum-signature-scheme SPHINCS+

For a seamless experience, consider using a dedicated console for these operations.

Prerequisites

  • Java-tron node fully synced with the TRON network
  • Post-quantum signature scheme supported by the java-tron implementation
  • Correct configuration parameters for the post-quantum signature scheme
