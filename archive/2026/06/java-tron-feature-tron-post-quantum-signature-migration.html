TRON Post-Quantum Signature Migration Failure

Symptom

  • Users encounter issues when attempting to migrate to post-quantum signatures on the TRON network, resulting in failed transactions and error messages indicating a migration failure.

Technical Root Cause

  • The error occurs due to incomplete or incorrect configuration of the post-quantum signature parameters, leading to a mismatch between the expected and actual signature formats.

Manual Overrides

  • Update the Tron configuration file to include the post-quantum signature parameters:
  • Set the signatureScheme to SPHINCS+
  • Configure the postQuantum settings with the correct parameters
  • Verify that the tron.jar file is updated to the latest version
{
  "signatureScheme": "SPHINCS+",
  "postQuantum": {
    "enabled": true,
    "parameters": {
      "treeHeight": 10,
      "wotsPlus": true
    }
  }
}

Consider utilizing an external console for further debugging and configuration.

Prerequisites

  • The latest version of Java-Tron installed
  • A compatible TRON wallet with post-quantum signature support
  • A thorough understanding of TRON's post-quantum signature migration process
