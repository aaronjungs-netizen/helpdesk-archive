TRON Post-Quantum Signature Migration Failed

Symptom

  • Users encounter an error message indicating that the TRON post-quantum signature migration has failed, resulting in failed transactions and inability to access certain features on the TRON network.

Technical Root Cause

  • The error occurs due to the incomplete or incorrect migration of post-quantum signatures, which are required for secure transactions on the TRON network. This can be caused by outdated software, incorrect configuration, or incomplete data migration.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Update the TRON network configuration to ensure compatibility with post-quantum signatures
  • Run a manual migration script to update the signature schemes
  • Verify the integrity of the migrated data using the following command:
{
  "migration": {
    "postQuantum": true,
    "signatureScheme": "SPHINCS"
  }
}

Consider using an external console, such as TronWeb or TronBox, to execute the migration commands.

Prerequisites

  • Java-Tron node installed and running
  • TRON network configuration updated to support post-quantum signatures
  • Backup of the wallet data and transaction history
