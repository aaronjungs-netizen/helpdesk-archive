TRON Post-Quantum Signature Migration Failed

Symptom

  • Users experience a failure in migrating to post-quantum signatures on the TRON network, resulting in an inability to complete transactions or interact with smart contracts.

Technical Root Cause

  • The error occurs due to a mismatch between the expected and actual cryptographic libraries used for post-quantum signature migration, leading to a compatibility issue within the java-tron implementation.

Manual Overrides

  • Update the cryptographic library to the latest version compatible with post-quantum signatures.
  • Configure the java-tron node to use the new post-quantum signature scheme by default.
  • Restart the java-tron node with the updated configuration, using the following command:
java -jar java-tron.jar --post-quantum-signature-enabled=true

Consider using an external console for more complex configuration adjustments.

Prerequisites

  • Ensure the latest version of java-tron is installed.
  • Verify the compatibility of the post-quantum signature library with the TRON network.
  • Backup existing configurations and data before attempting the migration.
