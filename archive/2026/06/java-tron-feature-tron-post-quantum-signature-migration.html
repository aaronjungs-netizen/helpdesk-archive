Java Tron Feature Tron Post Quantum Signature Migration

Symptom

  • Users encounter issues with transaction signatures, resulting in failed transactions, and observe error messages indicating incompatible signature schemes.

Technical Root Cause

  • The error occurs due to the migration from traditional elliptic curve cryptography to post-quantum signatures, which are not backward compatible, causing transactions signed with the old scheme to be rejected by the network.

Manual Overrides

  • To resolve the issue, users can attempt the following steps:
  • Update their wallet software to the latest version supporting post-quantum signatures.
  • Use the following command to manually configure the signature scheme: bash java -jar java-tron.jar --signature.scheme post_quantum
  • Verify that their TRON node is updated and configured correctly to handle post-quantum transactions.

For a smooth migration, ensure you have: * The latest version of Java-Tron installed * A compatible TRON node updated to support post-quantum signatures * Sufficient bandwidth and resources for the migration process

Consider utilizing an external console for more complex configurations.
