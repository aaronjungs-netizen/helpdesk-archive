Metamask Extension Bug Networks Miss Match Between Network Pick

Symptom

  • Users experience a mismatch between the selected network in the Metamask extension's network picker and the network displayed in the modal, showing "All popular" networks, which can lead to incorrect transaction routing and potential losses.

Technical Root Cause

  • The error occurs due to a desynchronization between the Metamask extension's internal network state and the network configuration stored in the browser's local storage, often caused by a recent update or manual network configuration changes.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Reset the Metamask extension's network configuration to its default state
  • Update the Metamask extension to the latest version
  • Manually configure the network settings using the following JSON configuration:
{
  "networks": {
    "1": {
      "name": "Mainnet",
      "chainId": 1,
      "currency": "ETH"
    }
  }
}

Consider utilizing an external console for a more streamlined debugging experience.

Prerequisites

  • Metamask extension installed and configured
  • Basic understanding of Ethereum network configurations
  • Latest version of the browser and Metamask extension
