Critical Trc 20 Energy Spike Error

Symptom

  • Users encounter a "Critical TRC-20 energy spike error" message when attempting to execute smart contracts or transfer tokens on the TRON network, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to an unexpected surge in energy consumption by TRC-20 smart contracts, exceeding the allocated energy limits and causing the transaction to fail.

Manual Overrides

  • Adjust the --energyLimit parameter in the TronWeb configuration to increase the energy limit for transactions.
  • Use the tronweb.transactionBuilder API to manually set the energy limit for specific transactions.
  • Update the TRON node configuration with the following settings:
{
  "energy": {
    "limit": 10000000
  }
}

Consider utilizing an external console for more advanced configuration options.

Prerequisites

  • TRON node version 4.5.0 or higher
  • TronWeb version 4.2.0 or higher
  • A basic understanding of TRON network configuration and smart contract development
