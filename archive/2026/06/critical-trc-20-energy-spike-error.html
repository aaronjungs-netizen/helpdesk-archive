Symptom

  • Users experience a "Critical TRC-20 energy spike error" when attempting to execute smart contracts or perform transactions on the TRON network, resulting in failed transactions and potential loss of assets.

Technical Root Cause

  • The error occurs due to an unexpected surge in energy consumption by TRC-20 tokens, exceeding the allocated energy limits for user accounts, causing the network to reject transactions to prevent overload.

Manual Overrides

  • Increase the energy limit for the user account using the tronweb interface.
  • Configure the TRON node to allow for dynamic energy allocation using the following code block:
{
  "energy": {
    "limit": 10000,
    "dynamic": true
  }
}
  • Utilize the tron-console command-line tool to manually adjust energy parameters for specific accounts.

Consider utilizing an external console for more advanced troubleshooting and configuration options.

Prerequisites

  • A basic understanding of the TRON network and its energy mechanics
  • Access to the TRON node configuration files
  • Familiarity with the tronweb and tron-console tools
