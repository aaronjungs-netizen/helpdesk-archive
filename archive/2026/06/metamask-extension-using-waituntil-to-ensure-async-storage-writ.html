Metamask Extension Using Waituntil To Ensure Async Storage Writes Finish

Symptom

  • Users encounter an issue where the Metamask extension is using waitUntil to ensure async storage writes finish, but this process seems to be stuck or timing out, resulting in an error message indicating that the extension is waiting for async storage writes to complete.

Technical Root Cause

  • The error occurs due to asynchronous storage operations not completing within the expected timeframe, causing the waitUntil function to timeout and resulting in the error message.

Manual Overrides

  • To resolve this issue, users can try the following manual overrides:
  • Increase the timeout value for the waitUntil function to allow more time for async storage writes to complete.
  • Use a different async storage method that does not rely on waitUntil.
  • Implement a retry mechanism for async storage writes to handle temporary failures.
# Example of increasing the timeout value
const timeout = 30000; // 30 seconds
await waitUntil(asyncStorageWriteComplete, timeout);
