Metamask Extension Using Waituntil To Ensure Async Storage Writes Finish

Symptom

  • Users encounter an issue where the Metamask extension is utilizing the waitUntil function to ensure asynchronous storage writes are completed, but this process appears to be timing out or causing delays.

Technical Root Cause

  • The error occurs due to the asynchronous nature of storage writes in the Metamask extension, where the waitUntil function is used to pause the execution of the code until the storage writes are finished, but it may not be properly configured or may be experiencing unexpected delays.

Manual Overrides

  • To resolve this issue, users can attempt the following manual overrides:
  • Increase the timeout value for the waitUntil function to allow more time for the storage writes to complete.
  • Use a different asynchronous handling mechanism, such as Promises or Observables, to manage the storage writes.
  • Implement a retry mechanism to handle any failed storage writes.
