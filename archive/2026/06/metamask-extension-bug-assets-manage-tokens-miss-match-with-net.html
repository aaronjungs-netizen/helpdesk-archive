Metamask Extension Bug Assets Manage Tokens Miss Match With Net

Symptom

  • Users experience a mismatch between the tokens listed in the Metamask extension's "Assets" section and the network filter, resulting in tokens being displayed incorrectly or not at all.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's token management system, which fails to correctly synchronize token listings with the selected network, leading to discrepancies in token visibility.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Reset the Metamask extension's token cache by deleting the tokenCache.json file.
  • Update the Metamask extension to the latest version to ensure bug fixes are applied.
  • Use the following command to manually refresh the token list:
mm-token-refresh --network mainnet

Consider utilizing an external console for more advanced troubleshooting and configuration options.

Prerequisites

  • Metamask extension installed and configured
  • ETH network selected
  • Token management permissions enabled
