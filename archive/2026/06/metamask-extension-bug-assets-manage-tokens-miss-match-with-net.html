Metamask Extension Bug Assets Manage Tokens Miss Match With Net

Symptom

  • Users encounter a mismatch between the tokens listed in the Metamask extension's "Manage Tokens" section and the network filter, resulting in incorrect token displays and potential transaction issues.

Technical Root Cause

  • The error occurs due to a desynchronization between the Metamask extension's internal token registry and the Ethereum network's actual token listings, often caused by outdated extension versions, incorrect network configurations, or corrupted token data.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Reset the Metamask extension to its default settings
  • Update the token list manually using the tokenlist.json file
  • Use the following command to update the token registry:
curl -X POST -H "Content-Type: application/json" --data '{"jsonrpc":"2.0","method":"eth_tokenRegistry_update","params":{},"id":1}' https://mainnet.infura.io/v3/YOUR_PROJECT_ID

Consider using an external console for more advanced troubleshooting and configuration.

Prerequisites

  • Metamask extension version 10.11.0 or later
  • Ethereum network connection (mainnet or testnet)
  • Basic understanding of JSON data structures and API requests
