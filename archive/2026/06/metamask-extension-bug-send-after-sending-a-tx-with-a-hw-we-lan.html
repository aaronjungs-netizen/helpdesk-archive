Metamask Extension Bug Send After Sending A Tx With A Hw We Lan

Symptom

  • Users experience a bug where, after sending a transaction using a hardware wallet, the Metamask extension redirects them to the Tokens page instead of the expected page.

Technical Root Cause

  • The error occurs due to a mismatch in the expected behavior of the Metamask extension when interacting with hardware wallets, leading to an incorrect redirect after a successful transaction.

Manual Overrides

  • Reset the Metamask extension to its default settings to resolve any configuration issues.
  • Update the hardware wallet firmware to the latest version to ensure compatibility with the Metamask extension.
  • Use the following JSON configuration to override the default redirect behavior:
{
  "redirect": "accounts",
  "hwWallet": true
}

Prerequisites: * Metamask extension version 10.1.0 or higher * Hardware wallet firmware version 2.4.0 or higher * Ethereum network connection stability For further troubleshooting, consider using an external console to inspect the extension's logs and behavior.
