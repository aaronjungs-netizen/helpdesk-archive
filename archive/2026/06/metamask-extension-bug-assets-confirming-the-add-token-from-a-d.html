Metamask Extension Bug Assets Confirming The Add Token From A Dapp Dialog

Symptom

  • Users are unable to confirm the addition of a token from a decentralized application (dapp) using the Metamask extension, as the "Add token" page keeps loading indefinitely.

Technical Root Cause

  • The issue arises due to a bug in the Metamask extension's handling of token addition requests from dapps, causing the confirmation dialog to malfunction and resulting in a loop of the "Add token" page.

Manual Overrides

  • Try resetting the Metamask extension by clearing its storage:
  • Delete the extension-data folder in the Metamask data directory
  • Remove the storage.json file
  • Run the following command to reset the extension:
metamask_reset --clear-storage

For a seamless experience, consider utilizing an external console to execute these commands.

Prerequisites

  • Metamask extension installed and enabled
  • A compatible Ethereum dapp with token addition functionality
  • The latest version of the Metamask extension updated
