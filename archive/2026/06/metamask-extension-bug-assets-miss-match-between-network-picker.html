Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users experience a mismatch between the assets displayed in the Metamask network picker and the modal window showing "All popular" assets, leading to confusion and potential errors in transactions.

Technical Root Cause

  • The error occurs due to a desynchronization between the network configuration and the asset listing, often resulting from outdated extension versions, incorrect network settings, or corrupted cache data.

Manual Overrides

  • Clear the Metamask extension cache and storage:
  • Update Metamask to the latest version
  • Reset the network configuration to default settings
  • Execute the following command in the terminal to clear cache: bash rm -rf ~/.config/MetaMask/ Consider using an external console for more complex debugging tasks.

Prerequisites

  • Metamask extension installed and configured
  • Basic understanding of Ethereum networks and asset management
  • Familiarity with terminal commands and JSON data formats
