Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users observe a discrepancy between the network selected in the Metamask extension's network picker and the assets displayed in the modal, which shows "All popular" assets, regardless of the chosen network.

Technical Root Cause

  • The error occurs due to a mismatch between the network ID stored in the Metamask extension's state and the network ID used to fetch assets, likely caused by a bug in the extension's code.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the extension's settings to default using the mm-reset command
  • Use the following JSON configuration to override the network settings:
{
  "networkVersion": "1",
  "chainId": 1,
  "network": "mainnet"
}

Consider utilizing an external console for further debugging.

Prerequisites

  • Metamask extension installed and configured
  • ETH network selected in the network picker
  • Latest version of the Metamask extension
