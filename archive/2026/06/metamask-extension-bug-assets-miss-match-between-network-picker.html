Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users experience a mismatch between the assets displayed in the network picker and the modal window, showing "All popular" assets, when using the Metamask extension on the Ethereum (ETH) ecosystem.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the network picker and modal window are not properly synchronized, resulting in a discrepancy between the displayed assets.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the Metamask settings to default using the following command:
mm-reset --default
  • Manually configure the network settings using a JSON configuration file

Consider using an external console for further debugging and customization.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum (ETH) ecosystem selected as the primary network
  • Basic understanding of JSON configuration files and command-line interfaces
