Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • The Metamask extension displays a mismatch between the selected network in the network picker and the assets shown in the modal, which is supposed to show "All popular" assets but instead shows assets from a different network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that causes the network picker and the asset modal to become desynchronized, resulting in the display of incorrect assets.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the Metamask extension settings to default
  • Use the following JSON configuration to manually override the network settings:
{
  "network": "mainnet",
  "chainId": 1,
  "assets": ["ETH", "DAI", "USDC"]
}

Consider utilizing an external console for more advanced troubleshooting and configuration.

Prerequisites

  • Metamask extension installed and enabled
  • Ethereum network selected in the network picker
  • Assets modal open and displaying incorrect assets
