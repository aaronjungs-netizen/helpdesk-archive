Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem may encounter a bug where there is a mismatch between the network picker and the modal window displaying "All popular" assets, leading to confusion and potential issues with asset management.

Technical Root Cause

  • The error occurs due to a discrepancy in the network configuration and the asset list retrieval process, causing the network picker to show a different set of assets than what is displayed in the modal window.

Manual Overrides

  • To troubleshoot this issue, users can try the following manual overrides:
  • Reset the Metamask extension to its default settings
  • Update the asset list manually by adding the missing assets
  • Use the following JSON configuration to override the default asset list:
{
  "network": "mainnet",
  "assets": [
    {
      "address": "0x...asset_address",
      "symbol": "ETH",
      "decimals": 18
    }
  ]
}

Consider using an external console to execute these commands for more flexibility.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum (ETH) network selected
  • Basic understanding of JSON configuration files
