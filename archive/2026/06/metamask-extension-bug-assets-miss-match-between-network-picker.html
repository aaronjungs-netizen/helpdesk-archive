Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem may encounter a bug where there's a mismatch between the assets displayed in the network picker and those shown in the modal, specifically when it shows "All popular" assets.

Technical Root Cause

  • This error occurs due to a desynchronization between the network configuration and the asset list update mechanism within the Metamask extension, potentially triggered by network changes or updates to the asset lists.

Manual Overrides

  • To troubleshoot this issue, users can attempt the following manual overrides:
  • Clear the browser cache and restart the browser to ensure a fresh start.
  • Check for updates to the Metamask extension and install the latest version.
  • Use the following JSON configuration to manually reset the asset list:
{
  "network": "mainnet",
  "assets": []
}

Consider utilizing an external debugging console for deeper inspection of the extension's behavior.

Prerequisites

  • Metamask extension installed and updated to the latest version.
  • A basic understanding of JSON configuration files.
  • Familiarity with browser cache management and extension updates.
