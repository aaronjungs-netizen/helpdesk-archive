Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users observe a mismatch between the network selected in the Metamask extension's network picker and the assets displayed in the modal, which shows "All popular" assets, potentially causing confusion and incorrect transactions on the Ethereum (ETH) network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that fails to update the asset list according to the chosen network, leading to a discrepancy between the expected and actual assets displayed.

Manual Overrides

  • Attempt to refresh the Metamask extension and retry the transaction.
  • Manually add the correct network and asset details in the extension's settings.
  • Utilize the following JSON configuration to override the default asset settings:
{
  "network": "mainnet",
  "assets": [
    {
      "address": "0x...asset_address",
      "symbol": "ETH",
      "decimals": 18
    }
  ]
}

Consider utilizing an external console for more granular control over network and asset configurations.

Prerequisites

  • Metamask extension installed and configured on the browser.
  • Ethereum (ETH) network selected in the extension's network picker.
  • The user has a basic understanding of blockchain and wallet interactions.
