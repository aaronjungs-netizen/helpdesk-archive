Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users experience a mismatch between the network selected in the Metamask extension's network picker and the assets displayed in the modal, which shows "All popular" assets instead of the expected assets for the chosen network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that causes the network picker and the asset modal to become desynchronized, resulting in the incorrect display of assets.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Clear the Metamask extension's cache and restart the browser
  • Update the Metamask extension to the latest version
  • Use the following JSON configuration to manually set the network and asset settings:
{
  "network": "mainnet",
  "assets": ["ETH", "DAI", "USDC"]
}

Consider utilizing an external console for more advanced troubleshooting and configuration.

Prerequisites

  • Metamask extension installed and configured
  • Latest version of the browser
  • Basic understanding of blockchain and cryptocurrency concepts
