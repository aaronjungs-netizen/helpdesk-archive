Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users experience a mismatch between the network selected in the Metamask extension's network picker and the assets displayed in the modal, which shows "All popular" assets from a different network.

Technical Root Cause

  • The error occurs due to a desynchronization between the network picker's state and the asset modal's data source, likely caused by a bug in the Metamask extension's code that handles network changes and asset retrieval.

Manual Overrides

  • Try resetting the Metamask extension by reinstalling it or clearing its storage.
  • Use the following JSON configuration to manually override the asset list:
{
  "network": "mainnet",
  "assets": [
    {
      "address": "0x...",
      "symbol": "ETH",
      "decimals": 18
    }
  ]
}
  • Utilize the eth_chainId RPC method to verify the current chain ID and ensure it matches the expected network.

Prerequisites: * Metamask extension installed and configured * Basic understanding of Ethereum networks and assets * Familiarity with JSON configuration files

For more advanced debugging, consider using an external console to inspect the Metamask extension's internal state.
