Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users observe a discrepancy between the assets listed in the Metamask network picker and those displayed in the modal window, which shows 'All popular' assets, even when a specific network is selected.

Technical Root Cause

  • The error occurs due to a mismatch in the internal state of the Metamask extension, where the network picker and the modal window do not synchronize correctly, leading to incorrect asset listings.

Manual Overrides

  • Try resetting the Metamask extension by deleting its local storage:
  • localStorage.clear() in the browser console
  • Manually removing the extension and reinstalling it
  • Using the following json payload to reset the extension's internal state:
{
  "network": "mainnet",
  "assets": []
}

For advanced troubleshooting, consider using an external console for more complex commands.

Prerequisites

  • Metamask extension installed and enabled
  • Ethereum network selected in the Metamask network picker
  • Latest version of the Metamask extension updated
