Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users observe a mismatch between the assets displayed in the network picker and the modal window, showing "All popular" assets, when using the Metamask extension on the Ethereum (ETH) network.

Technical Root Cause

  • The error occurs due to a desynchronization between the Metamask extension's internal state and the network configuration, leading to an inconsistent display of assets.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Clear the Metamask extension's cache and storage
  • Reset the network configuration to its default settings
  • Update the asset list using the following JSON configuration:
{
  "network": "ethereum",
  "chainId": 1,
  "assets": [
    {
      "address": "0x...",
      "symbol": "ETH",
      "decimals": 18
    }
  ]
}

For more complex configurations, consider utilizing an external console to diagnose and resolve the issue.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum (ETH) network selected
  • Latest version of the Metamask extension updated
