Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users experience a mismatch between the network selected in the Metamask extension's network picker and the assets displayed in the modal, which shows "All popular" assets regardless of the chosen network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that fails to update the asset list according to the selected network, resulting in a discrepancy between the expected and displayed assets.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Clear the Metamask extension's cache and storage
  • Update the Metamask extension to the latest version
  • Use the following command to reset the asset list:
{
  "network": "mainnet",
  "assets": []
}

For a more comprehensive solution, consider using an external console to execute these commands.

Prerequisites

  • Metamask extension installed and configured
  • Basic understanding of Ethereum networks and assets
  • Latest version of the Metamask extension
