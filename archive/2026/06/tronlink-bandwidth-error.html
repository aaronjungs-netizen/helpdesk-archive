Tronlink Bandwidth Error

Symptom

  • Users encounter an error message indicating a bandwidth issue when attempting to perform transactions or interactions on the TRON network using TronLink.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is a resource required for transactions on the TRON network. Bandwidth is depleted when users perform actions, and it can take time to replenish.

Manual Overrides

  • Increase the bandwidth by freezing TRX tokens, which allocates more bandwidth for transactions.
  • Use the tronweb API to check the current bandwidth and adjust accordingly:
tronwebtrx.getBalance().then(balance => console.log(balance));
  • Manually trigger bandwidth replenishment through the TronLink interface or using TRON's API.

Prerequisites: * TRX tokens available for freezing * TronLink wallet setup and configured * Basic understanding of TRON network resources

Consider utilizing an external console for more complex troubleshooting and direct interaction with the TRON network.
