Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network, resulting in failed transactions and inability to execute desired actions.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is a resource required for transactions on the TRON network. Bandwidth is typically obtained by freezing TRX (TRON's native cryptocurrency) or by purchasing it from other users.

Manual Overrides

  • Increase the bandwidth by freezing more TRX or purchasing bandwidth from other users.
  • Adjust the transaction parameters to require less bandwidth, such as reducing the transaction size or gas limit.
  • Utilize the following bash command to check available bandwidth:
curl -X POST \
  http://127.0.0.1:8090/wallet/getaccount \
  -H 'Content-Type: application/json' \
  -d '{"address": "your_tron_address"}'

For more detailed analysis, consider accessing the network via an external console to diagnose the issue more effectively.

Prerequisites

  • A TRON wallet with a sufficient balance of TRX.
  • A basic understanding of the TRON network and its transaction mechanics.
  • The TronLink extension installed and configured in the user's browser.
