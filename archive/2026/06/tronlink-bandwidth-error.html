Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network, resulting in failed or pending transactions.

Technical Root Cause

  • The error occurs due to insufficient bandwidth or energy on the user's TRON account, which is required to process transactions on the network.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens for a specified period.
  • Adjust the transaction's bandwidth and energy limits using the following code:
tronweb.transactionBuilder.freezeBalance(
  "10000000",  # bandwidth
  "0",         # duration (0 for permanent freeze)
  true         # frozen
);
  • Verify the account's bandwidth and energy levels using the TRON Web API to ensure they are sufficient for the transaction.

Consider using an external console for more advanced troubleshooting and debugging.

Prerequisites

  • TRON wallet with a sufficient balance of TRX tokens
  • Basic understanding of TRON's resource management system
  • Access to the TRON Web API or a compatible wallet interface
