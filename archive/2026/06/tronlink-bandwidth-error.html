Tronlink Bandwidth Error

Symptom

  • Users encounter an error message indicating a bandwidth issue while attempting to perform transactions or interact with smart contracts on the TRON network using TronLink.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is a resource required for transaction processing on the TRON network. Bandwidth is consumed when sending transactions, and if a user's account does not have enough bandwidth, transactions will fail.

Manual Overrides

  • Increase bandwidth by freezing TRX tokens, which allocates more bandwidth to the user's account.
  • Use the TRON Web3 API to check the current bandwidth points and adjust accordingly.
  • Execute the following command to unfreeze TRX and reclaim bandwidth:
tronweb.transactionBuilder.unfreezeBalance(tronweb.transactionBuilder.freezeBalance.unfreezeTRX);

Consider utilizing an external console for advanced debugging and transaction construction.

Prerequisites

  • A TRON-based wallet like TronLink.
  • A basic understanding of TRON's resource management system.
  • Sufficient TRX tokens for freezing to increase bandwidth.
