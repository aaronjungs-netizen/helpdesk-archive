Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network, resulting in failed transactions and inability to execute desired actions.

Technical Root Cause

  • The error occurs due to insufficient bandwidth on the user's TRON account, which is required to process transactions and interactions on the network. TRON's bandwidth system is designed to prioritize and manage network congestion, and insufficient bandwidth can lead to transaction failures.

Manual Overrides

  • Increase account bandwidth by freezing TRX tokens, which allocates bandwidth to the user's account.
  • Use the TRON Web Wallet to adjust bandwidth settings and prioritize transactions.
  • Utilize the following JSON-RPC command to manually adjust bandwidth:
{
  "jsonrpc": "2.0",
  "method": "triggerSmartContract",
  "params": [
    "TXID",
    "contract_address",
    {
      "function_selector": "freeze",
      "parameter": {
        "type": "freeze",
        "value": "10000000"
      }
    }
  ],
  "id": 1
}

Consider utilizing an external console for more advanced troubleshooting and manual overrides.

Prerequisites

  • TRON account with sufficient TRX balance
  • TRON Web Wallet or compatible wallet software
  • Basic understanding of TRON's bandwidth system and transaction processing
