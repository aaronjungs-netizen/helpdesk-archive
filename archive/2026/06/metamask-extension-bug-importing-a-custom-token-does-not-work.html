Metamask Extension Bug Importing A Custom Token Does Not Work

Symptom

  • Users are unable to import custom tokens into their Metamask wallet, with the import function failing to recognize or add the token.

Technical Root Cause

  • The error occurs due to incorrect token configuration, outdated Metamask extension, or insufficient permissions, preventing the wallet from successfully importing the custom token.

Manual Overrides

  • Check the token's smart contract address for accuracy and ensure it matches the one provided by the token's issuer.
  • Verify that the token's decimal places are correctly set, as incorrect decimal places can lead to import failures.
{
  "address": "0x...token_contract_address",
  "symbol": "...token_symbol",
  "decimals": ...token_decimal_places,
  "image": "...token_image_url"
}

Consider utilizing an external console to diagnose and resolve token import issues.

Prerequisites

  • Metamask extension is up-to-date
  • Custom token's smart contract address is correct
  • User has sufficient permissions to import custom tokens
