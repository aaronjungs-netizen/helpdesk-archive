Metamask Extension Bug Importing A Custom Token Does Not Work

Symptom

  • Users are unable to import custom tokens into their Metamask wallet, resulting in an unsuccessful token addition process.

Technical Root Cause

  • The error occurs due to incorrect token configuration or incompatible token standards, causing Metamask to fail in recognizing and importing the custom token.

Manual Overrides

  • Verify the token's contract address and ensure it matches the one provided by the token issuer.
  • Check the token's decimal places and symbol to ensure they are correctly configured.
  • Use the following JSON configuration to manually add the token:
{
  "address": "0x...token contract address",
  "symbol": "...token symbol",
  "decimals": ...token decimal places,
  "image": "...token image URL"
}

Consider utilizing an external console for advanced token configuration and troubleshooting.

Prerequisites

  • Metamask extension installed and configured
  • Custom token contract address and details available
  • Basic understanding of Ethereum token standards
