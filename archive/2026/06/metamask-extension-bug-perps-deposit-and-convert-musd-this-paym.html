Metamask Extension Bug Perps Deposit And Convert Musd This Payment Route Isn't Available

Symptom

  • Users encounter an error message stating "This payment route isn't available" when attempting to deposit and convert mUSD using the Metamask extension for Perps transactions on the Ethereum network.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the available liquidity routes and the user's requested transaction path, often resulting from outdated or incorrect configuration settings within the Metamask extension.

Manual Overrides

  • Attempt to refresh the transaction path by restarting the Metamask extension and retrying the transaction.
  • Manually configure the transaction route using the advanced settings:
{
  "routeConfig": {
    "mUSD": {
      "liquidityProvider": "updatedProvider",
      "conversionRate": 1.05
    }
  }
}
  • Clear the local storage and re-import the wallet to reset the configuration to its default state.

Prerequisites: - Ensure the latest version of the Metamask extension is installed. - Verify that the Ethereum network is properly configured and connected. - Check for any available updates to the wallet's firmware or software.

Consider utilizing an external console for more granular control over transaction configurations.
