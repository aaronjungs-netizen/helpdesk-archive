Metamask Extension Bug Perps Deposit And Convert Musd This Payment Route Isn't Available

Symptom

  • Users encounter an error message stating "This payment route isn't available" when attempting to deposit and convert mUSD using the Metamask extension for Perps.

Technical Root Cause

  • The error occurs due to a temporary mismatch in the available payment routes within the Metamask extension, potentially caused by a bug or outdated configuration.

Manual Overrides

  • Attempt to refresh the Metamask extension and retry the transaction
  • Check for any available updates for the Metamask extension and install the latest version
  • Use the following JSON configuration to manually override the payment route:
{
  "paymentRoutes": [
    {
      "from": "ETH",
      "to": "mUSD",
      "route": "UniswapV2"
    }
  ]
}
