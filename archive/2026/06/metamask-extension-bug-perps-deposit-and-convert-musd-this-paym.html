Metamask Extension Bug Perps Deposit And Convert Musd This Payment Route Isn't Available

Symptom

  • Users encounter an error message "This payment route isn't available" when attempting to deposit and convert mUSD using the Metamask extension for Perps.

Technical Root Cause

  • The error occurs due to a temporary mismatch in the liquidity pool or an issue with the payment routing configuration in the Metamask extension, preventing the conversion of mUSD.

Manual Overrides

  • Attempt to refresh the Metamask extension and retry the transaction.
  • Manually configure the payment route using the following JSON configuration:
{
  "paymentRoute": {
    "source": "mUSD",
    "destination": "ETH",
    "route": "UniswapV2"
  }
}
  • Utilize the ethers.js library to override the default payment routing and force the conversion.

Prerequisites: * Metamask extension installed and configured * ETH wallet balance sufficient for transaction fees * UniswapV2 protocol supported in the Metamask extension

Consider utilizing an external console to execute the override commands for more granular control over the transaction process.
