Metamask Extension Bug Perps Deposit And Convert Musd This Payment Route Isn't Available

Symptom

  • Users encounter an error message stating "This payment route isn't available" when attempting to deposit and convert mUSD using the Metamask extension for Perps.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the available liquidity pools and the payment routing configuration within the Metamask extension, preventing the desired transaction from being executed.

Manual Overrides

  • Attempt to refresh the Metamask extension and retry the transaction.
  • Check for updates to the Metamask extension and ensure the latest version is installed.
  • Utilize the following JSON configuration to manually specify the payment route:
{
  "paymentRoute": {
    "mUSD": {
      "destination": "0x...mUSD_contract_address",
      "source": "0x...user_wallet_address"
    }
  }
}

Consider utilizing an external console to diagnose and resolve the issue.

Prerequisites

  • Metamask extension installed and configured.
  • Sufficient ETH for transaction fees.
  • Latest version of the Metamask extension updated.
