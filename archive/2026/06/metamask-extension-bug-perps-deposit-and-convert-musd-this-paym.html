Metamask Extension Bug Perps Deposit And Convert Musd This Payment Route Isn't Available

Symptom

  • Users encounter an error message stating "This payment route isn't available" when attempting to deposit and convert mUSD using the Metamask extension for Perps.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the available liquidity routes on the Ethereum network and the payment paths configured within the Metamask extension, specifically affecting the mUSD conversion process.

Manual Overrides

  • Attempt to refresh the Metamask extension and retry the transaction.
  • Check for updates to the Metamask extension and ensure it is running the latest version.
  • Utilize a manual configuration override by inputting the following JSON configuration:
{
  "routeConfig": {
    "mUSD": {
      "conversionPath": ["USDT", "DAI"]
    }
  }
}

Consider utilizing an external console for deeper configuration adjustments.

Prerequisites

  • Ensure Metamask is updated to the latest version.
  • Verify sufficient ETH for gas fees.
  • Confirm the mUSD contract address is correctly configured in the Metamask extension.
