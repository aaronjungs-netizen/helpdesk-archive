Symptom

  • Users experience a bug when using the Metamask extension to sign a message, specifically when clicking the back button in the Bitcoin Snap Sign Message confirmation window. The error causes the signing process to malfunction, resulting in a failed transaction.

Technical Root Cause

  • The error occurs due to a compatibility issue between the Metamask extension and the Bitcoin Snap Sign Message module. When the back button is clicked, the signing process is interrupted, causing the transaction to fail.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the Bitcoin Snap Sign Message module settings
  • Use the following command to override the signing process:
mm-sign-message-override=true

Consider using an external console for further debugging.

Prerequisites

  • Metamask extension installed and configured
  • Bitcoin Snap Sign Message module enabled
  • Latest version of the Ethereum blockchain client installed
