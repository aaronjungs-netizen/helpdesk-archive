Metamask Extension Bug Sign Message Clicking Back Button In The

Symptom

  • Users experience a bug when attempting to sign a message using the Metamask extension on the Ethereum (ETH) ecosystem, specifically when clicking the back button during the Bitcoin Snap Sign Message confirmation process. The Metamask extension fails to handle this action correctly, resulting in an error.

Technical Root Cause

  • The error occurs due to a compatibility issue between the Metamask extension and the Bitcoin Snap Sign Message confirmation process. When the user clicks the back button, the extension is unable to properly cancel the sign message request, leading to a bug.

Manual Overrides

  • To troubleshoot this issue, users can try the following manual overrides:
  • Check for updates to the Metamask extension and ensure it is running the latest version.
  • Clear the browser's cache and cookies to resolve any potential conflicts.
  • Use the following JSON configuration to reset the sign message settings:
{
  "signMessage": {
    "confirmation": "required",
    "backButtonHandling": "cancel"
  }
}

Consider utilizing an external console for advanced troubleshooting and configuration.

Prerequisites

  • Metamask extension installed and configured on the browser
  • Latest version of the browser being used
  • Ethereum (ETH) ecosystem selected in the Metamask extension settings
