Metamask Extension Bug Sign Message Clicking Back Button In The

Symptom

  • Users experience a bug when using the Metamask extension to sign a message, specifically when clicking the back button in the Bitcoin Snap Sign Message confirmation dialog. The expected behavior is to return to the previous screen, but instead, the extension becomes unresponsive or displays an error message.

Technical Root Cause

  • The error occurs due to a miscommunication between the Metamask extension and the Bitcoin Snap Sign Message module. When the back button is clicked, the extension fails to properly handle the cancellation of the sign message request, resulting in a timeout or error.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Clear the Metamask extension's cache and storage
  • Reset the Bitcoin Snap Sign Message module's configuration
  • Use the following JSON configuration to override the sign message settings:
{
  "signMessage": {
    "timeout": 30000,
    "cancellationHandler": "manual"
  }
}

Consider using an external console to execute these commands for more flexibility.

Prerequisites

  • Metamask extension version 10.1.0 or later
  • Bitcoin Snap Sign Message module version 1.2.0 or later
  • Ethereum network connection established
