Symptom

  • The user encounters an error message indicating a failure in the block freeze operation due to a missing canonical hash, specifically "err="canonical hash missing, can't freeze block".

Technical Root Cause

  • This error occurs because the Ethereum node is unable to verify the canonical hash of a block, which is necessary for the freeze operation. This could be due to a corrupted database, outdated node software, or issues with the node's synchronization with the Ethereum network.

Manual Overrides

  • To resolve the issue, the following steps can be taken:
  • Stop the Ethereum node and restart it with the --reset flag to reset the node's database.

Prerequisites
