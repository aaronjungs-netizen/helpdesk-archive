Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users observe that the Stop Loss (SL) and Take Profit (TP) percentages in the Metamask extension unexpectedly change when modifying the leverage settings on the Ethereum (ETH) network.

Technical Root Cause

  • The issue arises due to a bug in the Metamask extension's handling of leverage adjustments, which inadvertently alters the SL and TP percentages.

Manual Overrides

  • To temporarily resolve the issue, try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the leverage settings to their default values
  • Apply a custom configuration using the following JSON code:
{
  "leverage": 1,
  "stopLoss": 5,
  "takeProfit": 10
}

Consider utilizing an external console for more advanced configuration options.
