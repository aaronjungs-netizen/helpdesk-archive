Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users experience unexpected changes in Stop-Loss (SL) and Take-Profit (TP) percentages when adjusting leverage levels in the Metamask extension on the Ethereum (ETH) ecosystem.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension's logic for handling leverage adjustments, which inadvertently alters the SL and TP percentages.

Manual Overrides

  • To temporarily resolve the issue, users can try the following manual overrides:
  • Adjust the leverage levels in small increments to minimize the impact on SL and TP percentages.
  • Use a separate calculator to determine the correct SL and TP percentages based on the desired leverage level.
  • Implement a custom script to override the Metamask extension's behavior, such as:
{
  "leverage": 5,
  "slPercentage": 10,
  "tpPercentage": 20
}

For more complex adjustments, consider using an external console for precise control.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum (ETH) ecosystem selected
  • Leverage levels adjusted in the Metamask extension settings
