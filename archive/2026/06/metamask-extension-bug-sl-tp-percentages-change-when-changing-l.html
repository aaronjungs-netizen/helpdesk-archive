Symptom

  • Users observe that their Stop Loss (SL) and Take Profit (TP) percentages are altered when they adjust the leverage in the Metamask extension on the Ethereum (ETH) ecosystem.

Technical Root Cause

  • The issue arises due to a bug in the Metamask extension's handling of leverage changes, which inadvertently modifies the SL and TP percentages, leading to unexpected trading outcomes.

Manual Overrides

  • To mitigate this issue, users can attempt the following manual overrides:
  • Edit the trading configuration to use fixed values instead of percentages for SL and TP.
  • Utilize a third-party trading tool that integrates with Metamask to manage trades.
  • Implement a custom script to monitor and adjust SL and TP percentages when leverage changes:
levchange.sh
#!/bin/bash
while true
do
  # Monitor leverage changes and adjust SL/TP percentages accordingly
  LEVERAGE=$(get_leverage)
  if [ $LEVERAGE -ne $PREV_LEVERAGE ]; then
    # Adjust SL/TP percentages
    set_sl_tp_percentages
  fi
  sleep 1
done
