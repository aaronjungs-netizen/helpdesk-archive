Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and frustration.

Technical Root Cause

  • The error occurs due to a timeout in the transfer loop, which can be caused by network congestion, insufficient gas, or faulty smart contract configuration, leading to a deadlock in the transaction processing.

Manual Overrides

  • Increase the gas limit for the transaction to ensure sufficient resources are allocated.
  • Verify the smart contract configuration to ensure it is correctly set up for jetton transfers.
  • Use the following command to override the default timeout settings:
tonos-cli call <contract_address> override_timeout '{"timeout": 300}'

Consider utilizing an external console for more advanced troubleshooting and configuration adjustments.

Prerequisites

  • Basic understanding of TON ecosystem and Tonkeeper functionality
  • Familiarity with smart contract configuration and gas management
  • Access to the tonos-cli command-line tool
