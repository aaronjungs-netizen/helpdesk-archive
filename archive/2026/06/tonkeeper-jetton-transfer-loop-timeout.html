Symptom

  • Users of the TON ecosystem may encounter an error message indicating a "Tonkeeper Jetton transfer loop timeout" when attempting to transfer jettons. This error manifests as a timeout in the transfer process, preventing the successful completion of the transaction.

Technical Root Cause

  • The Tonkeeper Jetton transfer loop timeout occurs due to issues with the transaction processing mechanism within the TON network. This can be caused by network congestion, incorrect configuration of the Tonkeeper wallet, or problems with the smart contract governing the jetton transfer.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Check the TON network for congestion and wait for a period of lower activity to retry the transfer.
  • Verify that the Tonkeeper wallet is properly configured and updated to the latest version.
  • Use the following command to manually initiate the transfer:
tonos-cli call <jetton_contract_address> transfer "{\"dest\":\"<recipient_address>\",\"value\":\"<amount>\"}" --abi <jetton_abi_file>

Consider utilizing an external console for more complex debugging and troubleshooting tasks.

Prerequisites

  • TON network account with sufficient balance
  • Tonkeeper wallet installed and configured
  • Latest version of tonos-cli installed on the system
