Symptom

  • Users of the TON ecosystem are experiencing a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, indicating a failure in the transaction process due to a timeout in the loop that manages the transfer.

Technical Root Cause

  • The error occurs due to a misconfiguration or overload in the Tonkeeper system, leading to an inability to complete the jetton transfer within the allocated time frame, thus resulting in a timeout.

Manual Overrides

  • To resolve the issue, try the following steps:
  • Increase the timeout value to allow more time for the transaction to complete.
  • Check the Tonkeeper node's health and ensure it is properly configured.
  • Use the following command to manually override the transfer loop:
tonos-cli call <jetton_address> transfer '{"dest":"<destination_address>","value":<amount>}' --timeout 60

Consider utilizing an external console for more complex troubleshooting and configuration adjustments.

Prerequisites

  • Familiarity with the TON ecosystem and its tools
  • Access to the Tonkeeper system and its configuration
  • Understanding of jetton transfer mechanisms in TON
