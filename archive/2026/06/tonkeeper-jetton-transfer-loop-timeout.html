Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and potential losses.

Technical Root Cause

  • The error occurs due to a timeout in the transfer loop, which can be caused by network congestion, slow transaction processing, or misconfigured settings in the Tonkeeper wallet.

Manual Overrides

  • Increase the transaction timeout limit to allow for longer processing times:
  • Update the Tonkeeper wallet configuration to prioritize transaction speed over security
  • Use the tonos-cli command to manually set the transaction timeout
  • Adjust the network settings to optimize for low-latency connections
tonos-cli config set timeout 300

Consider utilizing an external console for more advanced troubleshooting and configuration.

Prerequisites

  • Tonkeeper wallet installed and configured
  • Basic understanding of TON ecosystem and jetton transfers
  • tonos-cli tool installed and accessible
