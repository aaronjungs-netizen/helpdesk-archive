Symptom

  • The Tonkeeper Jetton transfer process is stuck in a loop, resulting in a timeout error, and the user is unable to complete the transfer.

Technical Root Cause

  • The error occurs due to a misconfiguration in the Tonkeeper settings or a temporary issue with the TON network, causing the transfer loop to exceed the maximum allowed time.

Manual Overrides

  • Check the Tonkeeper wallet configuration for any incorrect settings that may be causing the loop.
  • Verify the TON network status to ensure there are no ongoing issues that could be affecting the transfer.
  • Use the following command to reset the transfer loop:
tonos-cli call <wallet_address> reset_transfer_loop {}

Consider using an external console for more advanced troubleshooting capabilities.

Prerequisites

  • Tonkeeper wallet installed and configured
  • TON network account with sufficient funds
  • Basic understanding of TON blockchain and Tonos-cli commands
