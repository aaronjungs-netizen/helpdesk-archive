Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users encounter a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a prolonged loop in the transfer process, which exceeds the maximum allowed time limit, causing the transaction to timeout.

Manual Overrides

  • To resolve the issue, try the following overrides:
  • Increase the timeout limit for the transfer process
  • Verify the recipient's wallet address to ensure it is correct and compatible with Tonkeeper Jettons
  • Use a different transfer method, such as a manual transaction using the TON blockchain's raw transaction API
tonos-cli call <wallet_address> transferJetton "<recipient_address>" "<jetton_amount>" --timeout 300

Prerequisites: * Tonos-cli installed and configured on the system * A valid TON wallet with sufficient funds and Jettons * The recipient's wallet address and Jetton amount are known and correctly formatted Consider using an external console, such as a dedicated terminal, to execute the manual override commands.
