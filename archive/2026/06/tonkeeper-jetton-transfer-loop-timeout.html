Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper jetton transfer loop, which can be caused by network congestion, high gas fees, or issues with the smart contract.

Manual Overrides

  • Increase the gas limit to prioritize the transaction:
  • Update the transaction settings to use a higher gas price
  • Use a wallet with a built-in gas price adjustment feature
  • Implement a retry mechanism using the following code:
tonos-cli call <contract_address> retryTransfer "<jetton_name>" <recipient_address> --wallet <wallet_address>

Consider utilizing an external console for more advanced troubleshooting and debugging capabilities.

Prerequisites

  • Tonos-cli installed and configured on the system
  • A basic understanding of TON smart contracts and transaction mechanics
  • Access to the Tonkeeper wallet and contract addresses involved in the transfer loop
