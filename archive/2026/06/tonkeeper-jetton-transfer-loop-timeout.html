Symptom

  • The user encounters a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, resulting in a failed transaction and an unclear next step.

Technical Root Cause

  • This error occurs due to a timing issue in the Tonkeeper protocol, where the transfer loop exceeds the maximum allowed duration, causing the transaction to timeout and fail.

Manual Overrides

  • Increase the timeout duration by adjusting the transfer_timeout parameter in the Tonkeeper configuration.
  • Verify the network connection and ensure that it is stable to prevent interruptions during the transfer process.
  • Use the following JSON configuration to override the default settings:
{
  "transfer_timeout": 30000,
  "network_retry_count": 5
}

Consider utilizing an external console for further troubleshooting and configuration adjustments.

Prerequisites

  • Familiarity with Tonkeeper configuration and settings
  • Basic understanding of JSON formatting and syntax
  • Access to the Tonkeeper network and necessary credentials
