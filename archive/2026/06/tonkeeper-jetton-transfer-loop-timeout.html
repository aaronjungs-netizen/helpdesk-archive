Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, resulting in failed transactions and an inability to complete the intended action.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper Jetton transfer loop, which is likely caused by a slow or unresponsive network connection, high transaction volumes, or issues with the Tonkeeper smart contract.

Manual Overrides

  • Check the Ton network status for any ongoing maintenance or issues that may be affecting the transfer process.
  • Verify that the user's wallet is properly configured and has sufficient funds to complete the transaction.
  • Use the following JSON configuration to adjust the transfer loop timeout:
{
  "timeout": 30000,
  "retry_count": 5,
  "retry_delay": 5000
}

Consider utilizing an external console to diagnose and resolve the issue more efficiently.

Prerequisites

  • A basic understanding of the TON ecosystem and its components.
  • Familiarity with Tonkeeper and its jetton transfer functionality.
  • Access to the Ton network and a compatible wallet.
