Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the expected and actual transaction processing times on the TON blockchain, causing the transfer loop to exceed its designated timeout threshold.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the timeout threshold by modifying the tonkeeper.config file.
  • Restart the Tonkeeper application to refresh the transaction queue.
  • Verify the transaction status using the TON blockchain explorer.
{
  "timeout": 30000,
  "transaction_queue": {
    "max_size": 100,
    "refresh_interval": 1000
  }
}

Consider utilizing an external console for more advanced troubleshooting and configuration adjustments.

Prerequisites

  • Tonkeeper application installed and configured
  • Basic understanding of TON blockchain and transaction processing
  • Access to the TON blockchain explorer for transaction verification
