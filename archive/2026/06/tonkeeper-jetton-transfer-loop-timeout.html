Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, resulting in failed transactions and an inability to complete the desired action.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper system, which is designed to prevent infinite loops during jetton transfers. This can happen when there are issues with the smart contract, network congestion, or problems with the user's wallet configuration.

Manual Overrides

  • Check the smart contract code for any logical errors or infinite loop conditions.
  • Verify the user's wallet balance and ensure it has sufficient funds for the transfer.
  • Use the following command to override the default timeout settings:
tonos-cli call <contract_address> override_timeout '{"timeout": 300}'

Consider utilizing an external console for more advanced debugging and configuration options.

Prerequisites

  • Tonos-cli installed and configured on the system
  • A basic understanding of TON blockchain and smart contract functionality
  • Access to the contract address and wallet credentials
