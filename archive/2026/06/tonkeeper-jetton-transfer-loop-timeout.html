Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, resulting in failed transactions and uncertainty about the status of their assets.

Technical Root Cause

  • The error occurs due to a mismatch between the expected and actual transaction processing times, leading to a timeout in the Tonkeeper system. This discrepancy can be caused by network congestion, high transaction volumes, or issues with the Tonkeeper node synchronization.

Manual Overrides

  • Adjust the transaction timeout parameter to a higher value to accommodate slower processing times:
  • Increase the timeout threshold using the tonos-cli command
  • Verify the current network congestion and transaction queue length
  • Update the Tonkeeper configuration file with the new timeout value, as shown in the following code block:
{
  "timeout": 30000,
  "node": {
    "url": "https://main.ton.dev",
    "sync": true
  }
}

Consider utilizing an external console for more advanced debugging and configuration adjustments.

Prerequisites

  • Tonos-cli installed and configured on the system
  • Basic understanding of TON blockchain and Tonkeeper functionality
  • Access to the Tonkeeper configuration files and settings
