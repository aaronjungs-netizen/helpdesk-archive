Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a mismatch between the expected and actual transaction processing times, often caused by network congestion or incorrect configuration of the transfer loop.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Increase the transfer loop timeout threshold to account for network delays
  • Verify the correctness of the transfer loop configuration to ensure proper transaction processing
  • Use the following code to manually adjust the transfer loop settings:
{
  "transfer_loop": {
    "timeout": 30000,
    "retry_count": 5
  }
}

Consider using an external console for more advanced configuration options.

Prerequisites

  • Familiarity with TON ecosystem and Tonkeeper Jetton transfers
  • Basic understanding of network configuration and transaction processing
  • Access to the transfer loop configuration settings
