Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a mismatch between the expected and actual transfer times, causing the system to timeout before the transaction is confirmed on the TON blockchain.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the transfer timeout threshold to allow for longer transaction processing times.
  • Verify the TON blockchain network congestion and adjust the transfer schedule accordingly.
  • Use the following JSON configuration to update the transfer settings:
{
  "transfer_timeout": 300,
  "ton_network_congestion_threshold": 0.5
}

Consider utilizing an external console for more advanced troubleshooting and configuration adjustments.

  • Access to Tonkeeper API documentation
  • Basic understanding of JSON configuration files
