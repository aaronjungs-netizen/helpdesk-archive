Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users encounter a timeout error when attempting to transfer Jettons using Tonkeeper, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a prolonged loop in the transfer process, often caused by network congestion, insufficient gas, or incorrect configuration of the transfer parameters.

Manual Overrides

  • To resolve the issue, try the following steps:
  • Increase the gas limit to ensure sufficient resources for the transaction.
  • Verify the recipient's address and the Jetton configuration to prevent incorrect transfers.
  • Use a custom configuration to bypass the default transfer loop:
{
  "transfer": {
    "gas": 100000,
    "loopTimeout": 30000
  }
}

Consider utilizing an external console for more advanced configuration options.
