Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users encounter a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a synchronization issue between the Tonkeeper wallet and the TON blockchain, causing the transfer loop to exceed the maximum allowed time, thus timing out.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the timeout threshold using the tonos-cli command
  • Verify the wallet's connection to the TON blockchain using tonos-cli config
  • Reset the transfer loop by executing the following code:
tonos-cli call -s <wallet_address> --abi TonkeeperWallet.abi resetTransferLoop {}

Consider utilizing an external console for a more streamlined debugging experience.

Prerequisites

  • Tonos-cli installed and configured on the system
  • Tonkeeper wallet set up and connected to the TON blockchain
  • Basic understanding of TON blockchain and Tonkeeper wallet functionality
