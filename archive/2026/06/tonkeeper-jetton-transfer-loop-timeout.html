Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and potential loss of assets.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper transfer loop, which is responsible for facilitating jetton transfers between users. This timeout can be caused by network congestion, high transaction volumes, or issues with the Tonkeeper protocol.

Manual Overrides

  • Increase the timeout threshold to allow for longer transaction processing times:
  • Update the Tonkeeper configuration to extend the transfer loop timeout
  • Utilize a custom Tonkeeper client with optimized transfer settings
  • Implement a retry mechanism to reattempt failed transactions
tonos-cli config set timeout 300

Consider using an external console for advanced debugging and configuration.

Prerequisites

  • Basic understanding of the TON ecosystem and Tonkeeper protocol
  • Access to the Tonkeeper configuration files
  • Familiarity with the tonos-cli command-line tool
