Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users encounter a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the Tonkeeper wallet's internal state and the TON blockchain's current state, causing the transfer loop to timeout.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Restart the Tonkeeper wallet application to re-sync with the TON blockchain.
  • Check for any pending transactions and cancel them if necessary.
  • Use the following JSON-RPC command to manually update the wallet's internal state:
{
  "jsonrpc": "2.0",
  "method": "ton_updateWallet",
  "params": {
    "wallet_id": "user_wallet_id"
  }
}

Consider using an external console for more advanced troubleshooting and flexibility.

Prerequisites

  • Tonkeeper wallet application installed and configured.
  • Active internet connection for syncing with the TON blockchain.
  • Basic understanding of JSON-RPC commands and TON blockchain architecture.
