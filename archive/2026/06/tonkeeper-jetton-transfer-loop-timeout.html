Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper Jetton transfer loop, which can be caused by network congestion, high gas fees, or issues with the Tonkeeper smart contract.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the gas limit for the transaction to ensure it is processed quickly
  • Reduce the number of jettons being transferred to minimize the load on the network
  • Use a custom RPC endpoint to bypass potential issues with the default Tonkeeper node
