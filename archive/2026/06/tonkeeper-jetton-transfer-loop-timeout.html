Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the desired action.

Technical Root Cause

  • The error occurs due to a prolonged loop in the transaction processing mechanism, which exceeds the predefined timeout threshold, often caused by network congestion, high gas fees, or issues with the smart contract.

Manual Overrides

  • To resolve the issue, try the following:
  • Increase the gas limit to allow for more complex transactions: tonos-cli call <contract_address> setGasLimit <new_gas_limit> --key <key_file> bash tonos-cli config --url https://main.ton.dev
  • Restart the Tonkeeper application to reset the transaction queue
  • Verify the smart contract code for any potential bugs or inefficiencies

Prerequisites: * Tonos-cli installed and configured * Access to the contract address and key file * Basic understanding of TON blockchain and smart contracts

Consider utilizing an external console for advanced debugging and troubleshooting.
