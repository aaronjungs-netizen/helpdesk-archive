Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, resulting in failed transactions and potential losses.

Technical Root Cause

  • The error occurs due to a misconfiguration in the Tonkeeper wallet settings, causing an infinite loop of transfer requests, ultimately leading to a timeout.

Manual Overrides

  • Check the wallet's transaction history to identify any recurring patterns or stuck transactions.
  • Update the Tonkeeper wallet to the latest version to ensure compatibility with the TON network.
  • Use the following JSON configuration to override the default transfer loop settings:
{
  "transfer": {
    "loopTimeout": 30000,
    "retryAttempts": 5
  }
}

Consider utilizing an external console to gain more insight into the issue for a more detailed analysis.

Prerequisites

  • Tonkeeper wallet installed and configured on the user's device.
  • Active internet connection for TON network synchronization.
  • Basic understanding of JSON configuration files and their application in the Tonkeeper wallet.
