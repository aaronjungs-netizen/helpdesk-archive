Metamask Extension Bug Activity Contract Deployments Are Displayed As Contract Interaction

Symptom

  • Users of the Metamask extension may notice that contract deployments are incorrectly displayed as contract interactions, leading to confusion and potential mismanagement of on-chain activities.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's activity tracking module, which fails to accurately distinguish between contract deployment transactions and regular contract interaction transactions.

Manual Overrides

  • To troubleshoot this issue, users can attempt the following manual overrides:
  • Clear the Metamask extension's cache and storage
  • Reset the activity tracking module by running the following command: bash metamask --reset-activity-module
  • Manually update the Metamask extension to the latest version

Consider utilizing an external Ethereum blockchain console for advanced debugging.

Prerequisites

  • Metamask extension installed and configured
  • Basic understanding of Ethereum blockchain and smart contract interactions
  • Familiarity with command-line interfaces and debugging tools
