Metamask Extension Bug Activity Contract Deployments Are Displayed

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem notice that contract deployments are incorrectly displayed as contract interactions within the activity section of the extension.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's activity tracking and display logic, where it fails to differentiate between contract deployments and interactions, leading to misclassification and incorrect display of these activities.

Manual Overrides

  • To temporarily resolve this issue, users can try the following steps:
  • Clearing the Metamask extension's cache and storage to ensure a fresh start.
  • Disabling and re-enabling the extension to reset its internal state.
  • Editing the extension's configuration to filter out incorrect activity types, which can be achieved by adding the following JSON configuration:
{
  "activityFilters": {
    "contractDeployments": true,
    "contractInteractions": false
  }
}

Consider utilizing an external console for more complex configuration adjustments.

Prerequisites

  • Metamask extension installed and configured on a compatible web browser.
  • Basic understanding of Ethereum contract interactions and deployments.
  • Access to the Metamask extension's settings and configuration options.
